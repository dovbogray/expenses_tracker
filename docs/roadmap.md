
# Roadmap проєкту Expense Tracker

План розробки на найближчі тижні.

## Тиждень 3: Моделювання домену (DDD)
*   Аналіз предметної області (витрати, категорії, гаманець).
*   Створення глосарію термінів (Ubiquitous Language).
*   Реалізація класів сутностей `Expense` у коді (відокремлення бізнес-логіки від БД).

## Тиждень 4: API Контракт та Валідація
*   Опис REST API за допомогою OpenAPI (Swagger).
*   Реалізація повноцінного CRUD (Create, Read, Update, Delete) для витрат.
*   Валідація вхідних даних на рівні DTO.

## Тиждень 5: Стійкість та Обробка помилок
*   Впровадження єдиного формату помилок (RFC 7807).
*   Додавання механізму ідемпотентності (захист від подвійного списання/створення).
*   Налаштування логування запитів.

## Тиждень 6-7: Інфраструктура
*   Контейнеризація додатку (Docker).
*   Налаштування CI/CD пайплайнів (GitHub Actions) для автоматичного тестування.
