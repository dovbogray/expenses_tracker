# Business Requirements Document (BRD) - Expense Tracker

## 1. Назва та опис системи
**Назва:** Expense Tracker (Трекер витрат)

**Опис:**
Веб-застосунок для особистого обліку фінансів. Дозволяє користувачам швидко додавати витрати, переглядати історію транзакцій та аналізувати свої фінансові звички. Проєкт спрямований на вирішення проблеми неконтрольованих витрат шляхом надання простого та зручного інтерфейсу для фіксації покупок "на ходу".

## 2. Користувачі (Ролі)
*   **Гість (Guest):** Користувач без реєстрації. Має базовий доступ до функціонала (дані зберігаються локально або у демо-режимі).
*   **Зареєстрований користувач (User):** Має особистий акаунт. Може створювати, редагувати та видаляти свої витрати. Дані синхронізуються з базою даних.

## 3. Основні сценарії використання (Use Cases)
1.  **Додавання витрати:** Користувач вводить суму, опис та дату, натискає "Зберегти". Запис з'являється у списку.
2.  **Перегляд історії:** Користувач бачить список останніх витрат із сортуванням за датою.
3.  **Видалення запису:** Користувач може видалити помилково внесену витрату.

## 4. Функціональні вимоги
*   Система повинна мати REST API для CRUD операцій (Create, Read, Update, Delete) над витратами.
*   Кожна транзакція повинна містити поля: ID, сума, опис, категорія, дата.
*   Валідація даних: сума не може бути від'ємною, опис не може бути порожнім.

## 5. Нефункціональні вимоги
*   **Швидкодія:** Відповідь API повинна бути меншою за 200мс.
*   **Зручність:** Інтерфейс має бути адаптивним (зручним і на ПК, і на мобільному).
*   **Надійність:** Система повинна обробляти помилки (наприклад, відсутність зв'язку з БД) і повертати зрозумілі повідомлення.

## 6. Обмеження
*   На етапі MVP використовується база даних SQLite (локально).
*   Відсутня складна система аналітики/графіків (реалізується в майбутніх версіях).
*   Інтерфейс доступний лише українською мовою.
