# Context Map (Карта контекстів)

Система "Expense Tracker" розділена на наступні обмежені контексти (Bounded Contexts):

## 1. Accounting Context (Ядро)
Відповідає за фіксацію фінансових операцій.
*   **Ключові сутності:** Transaction (Транзакція), Category (Категорія).
*   **Відповідальність:** Валідація суми, перевірка дати, збереження опису.

## 2. User Identity Context (Підтримуючий)
Відповідає за ідентифікацію власника гаманця.
*   **Ключові сутності:** User (Користувач).
