# 1. Вибір архітектурного стилю

Date: 2025-12-17
Status: Accepted

## Context
Для успішного старту проекту 'Трекер витрат' пріоритетом є мінімізація початкової інфраструктурної складності заради високої швидкості розробки (Time-to-Market). При цьому обрана архітектура повинна залишатися гнучкою для еволюційного масштабування в майбутньому.

## Decision
Обрано архітектурний стиль **Модульний Моноліт (Modular Monolith)**.

## Consequences
* **Позитивні:** Спрощений процес розгортання (deployment), відсутність мережевих затримок між компонентами системи, легший рефакторинг та менше навантаження на інфраструктуру порівняно з мікросервісами.
* **Негативні:** Тісніша зв'язність коду (coupling), що вимагає суворої дисципліни при дотриманні меж модулів.

## Compliance
У разі зростання навантаження або розширення команди, окремі ізольовані модулі (наприклад, Auth або Reports) можуть бути винесені в окремі мікросервіси з мінімальними змінами в бізнес-логіці.
